## 🌱 このツールが解決する問題

Claude Code をよく使っていると、こんな瞬間ありませんか?

- **「昨日このエラー解決したはずなのに... またイチから説明?」**
- **「プロジェクト開始のたびにコンテキストを説明し直すの疲れた」**
- **「この作業3回目なのに、毎回最初からやり直しだ」**
- **「Claude が何が得意で何を間違えやすいか、私にも分からない」**

このフックをインストールすると、Claude Code が **自分で学習して記憶** し始めます。

| Claude がしてくれること | いつ |
|---|---|
| プロジェクトのスタックを検知 → 関連メモリを自動ロード | セッション開始時 |
| エラーが出たら過去の解決記録を自動マッチ | ビルド・コマンド失敗時 |
| 「このセッションで学んだこと保存しますか?」と聞いてくれる | セッション終了時 |
| 繰り返し作業を検知 → スキル化を提案 | 同じ作業を5回以上した時 |
| ドメイン別の精度を追跡 → 弱い領域を先んじて告白 | ビルドエラー検知時 |

**開発経験がなくても使えます。** ターミナルに1行コピペするだけ。

---

### ⭐ Memory-First ワークフロー (目玉機能)

入力: **「トーン見て」** — 他に文脈なし。

self-learn なし: 「どのトーン? テキストをください」と聞き返される。

self-learn あり: Claude が `~/.claude/memory/feedback_*tone*.md` を Glob → 該当ファイルを Read → 返答:

> 「公開ドキュメントは親切で前もって案内するトーンを好まれると記憶しています。その基準でチェックしますか? ファイルパスをください」
> (参照: feedback_oss_friendly_tone.md)

保存した好み・パターン・解決策があるすべてのトピックに機能 — **「メモリ確認して」と言う必要一切なし**。

## ⚡ 30秒インストール (一番簡単)

ターミナルを開いて、この1行をコピペ:

```bash
curl -sSL https://raw.githubusercontent.com/DAYLAB-HQ/claude-self-learn/main/quickstart.sh | bash
```

これだけ。jq インストール・ダウンロード・実行まで全自動。
(コードを先に確認したい方は下の手動インストールを。)

> **Windows**: 先に WSL (`wsl` コマンド) か Git Bash を開いてから同じ行をペーストしてください。

## 🚀 手動インストール (5分、コード確認したい方向け)

### Step 1: ターミナルを開く
- **Mac**: Spotlight (⌘+Space) で「ターミナル」を検索 → Enter
- **Linux**: ターミナルアプリを起動
- **Windows**: 少し手順が増えますが、できます!
  - **一番簡単**: WSL をインストール (Windows に組み込まれている「Linux モード」)。
    PowerShell を開いて `wsl --install` → 再起動 → スタートメニューから「Ubuntu」を起動。
    その中で下のコマンドが Mac/Linux と同じように動きます。
  - **別の方法**: [Git for Windows](https://git-scm.com/download/win) が既にあれば、
    スタートメニューから「Git Bash」を開いて下のコマンドを実行すれば大体動きます。

### Step 2: jq をインストール (初回のみ)
```bash
brew install jq
```

### Step 3: ダウンロード + インストール (1行)
```bash
git clone https://github.com/DAYLAB-HQ/claude-self-learn.git && cd claude-self-learn && ./install.sh
```

途中で **"Proceed? [Y/n]"** と聞かれたら **Y** を押して Enter。

### Step 4: 確認
Claude Code の新しいセッションで:
```
/self-learn:stats
```

ダッシュボードが表示されたらインストール成功! 🎉

## 🛡️ 安全にインストールされます

すでに Claude Code を使っている方でも **既存の設定は一切変更しません**。

### 絶対に上書きしないもの
- 既存のメモリファイル (`~/.claude/memory/`)
- 既存の学習ログ (`~/.claude/self-improvement/`)
- 既存のカスタムスキル・ルール・コマンド

### インストール前にプレビュー
```bash
./install.sh --dry-run
```
実際にインストールせず、**何がどこにインストールされるか** だけ見せてくれます。

### いつでもきれいにアンインストール
```bash
./uninstall.sh
```

## 📖 使い方

```
/self-learn

```

```

```
