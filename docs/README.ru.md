# self-learn

> **Небольшая система, которая позволяет Claude Code запоминать то, что он изучил — чтобы следующая сессия начиналась умнее предыдущей.**

## 🌱 Какие проблемы решает

Если вы регулярно пользуетесь Claude Code, вам знакомо:

- **«Я же вчера исправил эту ошибку... неужели надо объяснять заново?»**
- **«Устал каждый раз заново объяснять контекст проекта.»**
- **«Третий раз делаю одно и то же. Неужели Claude не может запомнить?»**
- **«Я сам не знаю, в каких областях Claude силён, а в каких часто ошибается.»**

Установите эти хуки, и Claude Code начнёт **учиться и запоминать сам**.

| Что делает Claude | Когда |
|---|---|
| Определяет стек проекта → подгружает связанные воспоминания | Начало сессии |
| Ищет предыдущие решения для знакомых ошибок | Build/команда упала |
| Спрашивает «Сохранить то, что узнал в этой сессии?» | Конец сессии |
| Предлагает создать skill для повторяющейся работы | Одна задача 5+ раз |
| Отслеживает точность по доменам → заранее отмечает слабые | Ошибки build обнаружены |

**Не нужно быть разработчиком, чтобы установить.** Просто вставьте команду в терминал.

---

### ⭐ Memory-First workflow (флагманская фича)

Ввод: **«проверь тон»** — без дополнительного контекста.

Без self-learn: Claude спрашивает «какой тон? кинь текст».

С self-learn: Claude делает `Glob` по `~/.claude/memory/feedback_*tone*.md`, `Read` совпадений и отвечает:

> «Помню — для публичных документов ты предпочитаешь дружелюбный, заранее поясняющий тон без защитных формулировок. Использовать этот стандарт? Пришли путь/текст.»
> (参照: feedback_oss_friendly_tone.md)

Работает для любой темы, по которой ты сохранял предпочтения/паттерны/решения — **никогда не надо говорить «посмотри в память»**.

## ⚡ Установка за 30 секунд (самый простой способ)

Открой терминал, вставь одну строку:

```bash
curl -sSL https://raw.githubusercontent.com/DAYLAB-HQ/claude-self-learn/main/quickstart.sh | bash
```

Всё. Скрипт сам поставит jq, скачает и запустит установщик.
(Хочешь сначала посмотреть код? См. ручную установку ниже.)

> **Windows**: Сначала открой WSL (`wsl`) или Git Bash, потом вставь эту строку.

## 🚀 Ручная установка (5 минут, с возможностью просмотреть код)

### Шаг 1: Открыть терминал
- **Mac**: Spotlight (⌘+Space) → найти "Terminal" → Enter
- **Linux**: Открыть приложение Terminal
- **Windows**: Пара дополнительных шагов, но это возможно!
  - **Самое простое**: установить WSL (встроенный в Windows «режим Linux»).
    Открой PowerShell, запусти `wsl --install`, перезагрузи ПК, запусти «Ubuntu» из меню Пуск.
    Внутри команды ниже работают так же, как на Mac/Linux.
  - **Альтернатива**: если у тебя уже установлен [Git for Windows](https://git-scm.com/download/win),
    открой «Git Bash» из меню Пуск — команды ниже должны работать.

### Шаг 2: Установить jq (один раз)
```bash
brew install jq
```

### Шаг 3: Скачать + установить (одна строка)
```bash
git clone https://github.com/DAYLAB-HQ/claude-self-learn.git && cd claude-self-learn && ./install.sh
```

Когда спросит **"Proceed? [Y/n]"** — нажмите **Y** и Enter.

### Шаг 4: Проверить
Откройте новую сессию Claude Code:
```
/self-learn:stats
```

Появился дашборд? Готово! 🎉

## 🛡️ Безопасная установка

Уже используете Claude Code? **Установщик не тронет вашу существующую конфигурацию.**

### Никогда не перезаписывается
- Ваши файлы памяти (`~/.claude/memory/`)
- Ваши логи обучения (`~/.claude/self-improvement/`)
- Ваши кастомные skills, rules и commands

### Просмотр перед установкой
```bash
./install.sh --dry-run
```

### Чистое удаление в любой момент
```bash
./uninstall.sh
```
